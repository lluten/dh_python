Git & GitHub

Version control is the safety net for your code. Learn how to save your history and collaborate.

Version Control

Git is a version control system that tracks changes in your code and lets you revert to earlier versions. Using Git and GitHub supports collaboration, reproducibility, and safe backup of your work.

The instructions provided are just one approach to configuring and utilizing Git. Because Git is highly adaptable, it supports a wide variety of workflows. For further details, please refer to the official Git documentation or GitHub documentation.

01. Preparation

To use GitHub, you need a GitHub account. You can create one at github.com/signup.

Additionally, you must install the Git software on your machine:

Download the installer from git-scm.com and run. You can click "Next" through most options, but we recommend selecting Use Visual Studio Code as Git's default editor if asked.

You can verify the installation in PowerShell:

POWERSHELL
git --version

Git needs to know who you are. This information is attached to every Commit you make.

POWERSHELL
# 1. Set your username
git config --global user.name "Your Name"

# 2. Set your email (must match your GitHub account)
git config --global user.email "you@example.com"

You can use Homebrew to install Git:

TERMINAL
brew install git

Alternatively, typing git in the terminal may trigger the Apple Command Line Tools installer.

02. SSH Keys

SSH keys allow you to securely connect to GitHub without entering your password every time.

Follow the GitHub documentation to generate and add an SSH key.

03. Connect to GitHub

Create a new repository

To create a new repository on GitHub, go to GitHub's new repository page. Make sure to select the Python template when creating a .gitignore file.

Clone the repository

After creating the repository on GitHub, you can clone it to your local machine using the SSH URL from your repository's page.

TERMINAL
git clone git@github.com:YourUsername/YourProject.git

Create a new repository

To create a new repository on GitHub, go to GitHub's new repository page.

Make sure to initialize the repository without a README or gitignore file.

Prepare your local project

Create a file in your project folder named .gitignore, starting with a dot, and with no other file extensions. This file tells Git which files or folders to ignore when tracking your changes.

Open the .gitignore file in a text editor and add the following line to ignore your virtual environment folder:

TEXT 
.venv/

Initialize Git and push to GitHub

Initialize Git in your local project, mark all files for tracking, commit them and push them to your GitHub repository, using your SSH URL.

TERMINAL
git init
git add . 
git commit -m "chore: init"
git branch -M main
git remote add origin git@github.com:YourUsername/YourProject.git
git push -u origin main

04. The Workflow

The basic Git workflow works in three steps:

Add (staging changes), Commit (saving changes locally), and Push (uploading changes to GitHub)

1. Add

Adding files means marking them for inclusion in the next commit (staging). You can either mark all files in your project folder using a . or select specific files.

CMD 
git add .

2. Commit

Committing files means saving them locally with a descriptive message. This creates a snapshot of your project at that moment.

Especially when working in a team, it's important to write clear and descriptive commit messages so that others can understand what changes were made. You can find a good convention for commit messages here.

CMD 
git commit -m "feat: add email notifications on new direct messages"

3. Push

Pushing files means uploading your local commits to GitHub.

CMD 
git push

05. Working with Branches (Advanced)

Branches are used to isolate work in progress from the main branch. This allows you to collaborate on different features without interfering with each other's work.
